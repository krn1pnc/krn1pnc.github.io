* B. Festivals in JOI Kingdom 2

我是计数菜逼．/ll

首先，正确的贪心应该是将区间按照右端点升序排序贪心．下文中称正确的贪心为算法 A，假贪心为算法 B．然后再称被算法 A 选中的区间为 A 区间，被算法 B 选中的区间为 B 区间，不会被任何算法选中的区间为 F 区间．

这个时候是不是应该观察一下性质！设我们有 nnn 个区间 [l1,r1],[l2,r2],⋯[ln,rn][l_1, r_1], [l_2, r_2], \cdots [l_n, r_n][l1​,r1​],[l2​,r2​],⋯[ln​,rn​]，A 区间的下标序列为 a1,a2,⋯ ,apa_1, a_2, \cdots, a_pa1​,a2​,⋯,ap​，B 区间的下标序列为 b1,b2,⋯ ,bqb_1, b_2, \cdots, b_qb1​,b2​,⋯,bq​，由于我们的贪心是对的，所以有 p≥qp \ge qp≥q．

先来看算法 A，由于算法 A 按右端点升序贪心，可以观察到：

  1. 不存在 iii，使得 ri<ra1r_i < r_{a_1}ri​<ra1​​．
  2. 不存在 i,ji, ji,j，使得 raj<lir_{a_j} < l_iraj​​<li​ 且 ri<raj+1r_i < r_{a_{j + 1}}ri​<raj+1​​．

对于算法 B，类似地有：

  1. 不存在 iii，使得 li<lb1l_i < l_{b_1}li​<lb1​​．
  2. 不存在 i,ji, ji,j，使得 rbj<lir_{b_j} < l_irbj​​<li​ 且 li<lbj+1l_i < l_{b_{j + 1}}li​<lbj+1​​．

显然，这是充要的．

然后我们尝试解决这个计数问题，发现直接计数很难，但是总方案数是可以算的，所以应当计算 p=qp = qp=q 的方案数．

这时候需要巨量观察，我先从官方题解中贺一张图：

其中红色区间为 A 区间，蓝色区间为 B 区间，紫色区间同时为 A 区间和 B 区间，未被染色的区间为 F 区间．

可以发现，在某个 p=qp = qp=q 的选择方案中：

  1. lb1≤la1l_{b_1} \le l_{a_1}lb1​​≤la1​​，ra1≤rb1r_{a_1} \le r_{b_1}ra1​​≤rb1​​．否则不满足两个算法的条件 1．
  2. rai≤rbir_{a_i} \le r_{b_i}rai​​≤rbi​​．否则不满足算法 A 的条件 2．
  3. [lai,rai]∩[lbi,rbi]≠∅[l_{a_i}, r_{a_i}] \cap [l_{b_i}, r_{b_i}] \not= \varnothing[lai​​,rai​​]∩[lbi​​,rbi​​]=∅．否则 [lbi,rbi][l_{b_i}, r_{b_i}][lbi​​,rbi​​] 会被算法 A 选中，此时不满足 p=qp = qp=q．
  4. rbi−1<rair_{b_{i - 1}} < r_{a_i}rbi−1​​<rai​​，否则由于 rbi−1<lbir_{b_{i - 1}} < l_{b_i}rbi−1​​<lbi​​，不满足条件 3.

然后就可以开始设计 DP 了！我们大概需要设计一个这样的东西：每次插入一对 A、B 区间和若干 F 区间，然后在这一过程中计算方案数．

然后我们发现，如果我们按照两算法的选择顺序插入区间，为了满足上述所有条件，F 区间能够选择的左端点数会和前面的具体放置方案相关，必须要设计到状态里，做出一个类似线头 DP 的东西．但是 F 区间的右端点限制是非常松的：只需要在当前 A 区间右端点的右边．这启发我们倒序插入区间．

设 fi,0/1f_{i, 0 / 1}fi,0/1​ 表示插入了后 iii 个区间，上次插入的 A、B 区间是否不同的方案数．转移枚举插 jjj 个 F 区间，使得这 jjj 个 F 区间的左端点在当前插入的 A 区间的右端点右边，分类讨论：

  1. fi,1→fi+j+2,1f_{i, 1} \to f_{i + j + 2, 1}fi,1​→fi+j+2,1​．

    此时要分两类讨论：

    1. 当前插入的 B 区间右端点在上一次插入的 A 区间左端点的左边，如图：

      考虑 F 区间右端点的方案数：考虑逐个插入右端点，对于第 111 个插入的右端点，如图所示，可插入的范围中一共有 2(i−2)+12(i - 2) + 12(i−2)+1 个点，也就是有 2(i−2)+1+1=2i−22(i - 2) + 1 + 1 = 2i - 22(i−2)+1+1=2i−2 个区间可供插入，而插入后又会新增一个区间，故插入一共 jjj 个右端点的方案数为 (2i−2)j‾(2i - 2)^{\overline{j}}(2i−2)j​．

      再考虑安排 F 区间的左端点：由于右端点已经标号，所以左端点直接按序匹配就能不重不漏地统计所有方案，不用再次标号，而又由于 B 区间的条件 2，相邻 B 区间之间不能有左端点，那么方案数就相当于把 jjj 个球放入 333 个盒子中，盒子可空，即为 (j+3−13−1)=(j+22)\binom{j + 3 - 1}{3 - 1} = \binom{j + 2}{2}(3−1j+3−1​)=(2j+2​)．

      于是得到这种情况的转移：fi,1×(j+22)×(2i−2)j‾→fi+j+2,1f_{i, 1} \times \binom{j + 2}{2} \times (2i - 2)^{\overline{j}} \to f_{i + j + 2, 1}fi,1​×(2j+2​)×(2i−2)j​→fi+j+2,1​

    2. 当前插入的 B 区间右端点在上一次插入的 A 区间左端点的右边，如图：

      发现转移和上面那种情况是一样的，可以直接合到一起转移！

    于是得到这种情况的转移方程：

    fi,1×(j+2)2‾×(2i−2)j‾→fi+j+2,1f_{i, 1} \times (j + 2)^{\underline{2}} \times (2i - 2)^{\overline{j}} \to f_{i + j + 2, 1} fi,1​×(j+2)2​×(2i−2)j​→fi+j+2,1​

  2. fi,1→fi+j+1,0f_{i, 1} \to f_{i + j + 1, 0}fi,1​→fi+j+1,0​．

    转移和 1 情况的差别只有左端点能够选择的区间数目变成了 222，故转移方程为：

    fi,1×(j+1)×(2i−2)j‾→fi+j+1,0f_{i, 1} \times (j + 1) \times (2i - 2)^{\overline{j}} \to f_{i + j + 1, 0} fi,1​×(j+1)×(2i−2)j​→fi+j+1,0​

  3. fi,0→fi+j+2,1f_{i, 0} \to f_{i + j + 2, 1}fi,0​→fi+j+2,1​．

    此时左端点只有 222 个区间可选，且右端点能选择的区间内只有 2(i−1)=2i−22(i - 1) = 2i - 22(i−1)=2i−2 个端点，故转移方程为：

    fi,0×(j+1)×(2i−1)j‾→fi+j+2,1f_{i, 0} \times (j + 1) \times (2i - 1)^{\overline{j}} \to f_{i + j + 2, 1} fi,0​×(j+1)×(2i−1)j​→fi+j+2,1​

  4. fi,0→fi+j+1,0f_{i, 0} \to f_{i + j + 1, 0}fi,0​→fi+j+1,0​．

    左端点只有 111 个区间可选，且右端点能选择的区间内只有 2i−22i - 22i−2 个端点，故转移方程为：

    fi,0×(2i−1)j‾→fi+j+1,0f_{i, 0} \times (2i - 1)^{\overline{j}} \to f_{i + j + 1, 0} fi,0​×(2i−1)j​→fi+j+1,0​

然后发现还有 F 区间左端点在第一个 A 区间右端点左边的方案没有统计，类似讨论即可．

时间复杂度 O(n2)O(n ^ 2)O(n2)，在 loj 上能过．正解还要上个半在线卷积．

* C. Passport

可以注意到，由于签发国家总是被对应的可达区间包含，于是可达所有点等价于可达 111 和 nnn．

如果只用到达一边可以直接线段树优化建边跑最短路解决，但是现在要求到达两边的最小代价．有一个朴素的想法是，对点 iii 分别求出到 111 的代价 c1,ic_{1, i}c1,i​ 和到 nnn 的代价 cn,ic_{n, i}cn,i​，然后答案就是 c1,i+cn,ic_{1, i} + c_{n, i}c1,i​+cn,i​．这显然是错的，因为两段路径可能有交．

然后就不会了啊！看题解发现最优方案一定形如起点先到达某个中继点，然后再从中继点前往 111 和 nnn，然后就先将每一个点的答案初始化成 c1,i+cn,ic_{1, i} + c_{n, i}c1,i​+cn,i​，再跑一遍 01 bfs 松弛掉重复的部分．

* C. Mizuyokan 2

好牛逼．

首先发现，对于合法的一个划分，一定能够通过调整划分方案，使得作为谷的连续段长度为 111．

然后可以发现，如果我们划分出了若干段 [li,ri][l_i, r_i][li​,ri​]，使得对于所有 iii，有 ∑j=liriai>max⁡{ali−1,ari+1}\sum\limits_{j = l_i}^{r_i} a_i > \max\{a_{l_i - 1}, a_{r_i + 1}\}j=li​∑ri​​ai​>max{ali​−1​,ari​+1​}，那么一定可以在此基础上调整出一个满足题意方案．显然这种划分和题目中给定的划分等价，拥有同样的段数上界．

下文中称将一个数选出来作为某个 ri+1r_i + 1ri​+1 为选择这个数．由于要划分出尽量多的段，所以我们希望下一个选中的位置距离当前位置尽量近．设 nxti\mathrm{nxt}_inxti​ 为下一个选择的位置，再记 prei\mathrm{pre}_iprei​ 为最大的 jjj，使得 ∑k=j+1i−1ak>max⁡{aj,ai}\sum\limits_{k = j + 1}^{i - 1} a_k > \max\{a_j, a_i\}k=j+1∑i−1​ak​>max{aj​,ai​}，那么 nxti\mathrm{nxt}_inxti​ 就是最小的 jjj，使得 prej≥i\mathrm{pre}_j \ge iprej​≥i．求出 nxt\mathrm{nxt}nxt 后倍增就能过掉不带修改的包．

考虑带修改怎么做，这个时候有牛逼结论：

引理 1：nxti−i\mathrm{nxt}_i - inxti​−i 是 O(log⁡V)O(\log V)O(logV) 级别的．

证明：

令 c=i+⌊log⁡V⌋+2c = i + \lfloor \log V \rfloor + 2c=i+⌊logV⌋+2．考虑反证法，令该结论不成立，那么对于所有 i≤j<ci \le j < ci≤j<c，都有 aj≤∑k=j+1c−1aka_j \le \sum\limits_{k = j + 1}^{c - 1} a_kaj​≤k=j+1∑c−1​ak​，归纳一下可以得出，ai≥2c−1−iac≥2c−1−i≥2⌊log⁡V⌋+1>Va_i \ge 2^{c - 1 - i} a_c \ge 2^{c - 1 - i} \ge 2^{\lfloor \log V \rfloor + 1} > Vai​≥2c−1−iac​≥2c−1−i≥2⌊logV⌋+1>V，矛盾．

然后就可以在线段树上维护每个区间前 O(log⁡V)O(\log V)O(logV) 个位置在区间内能够跳到的最远位置和步数．每次修改暴力重算有用的 pre\mathrm{pre}pre 和 nxt\mathrm{nxt}nxt 后重建线段树即可．时间复杂度 O(nlog⁡nlog⁡V)O(n \log n \log V)O(nlognlogV)．

* A. Chorus

感谢 这篇题解，感觉比官方题解更清晰！

将 A 看作 (\texttt{(}(，B 看作 )\texttt{)})，题意就是经过最小次数的交换，使得能够划分出 kkk 个子序列，让每个子序列都形如 (((⋯)))\texttt{(((} \cdots \texttt{)))}(((⋯)))．

然后不会做了！看题解得知，无论如何划分，第 iii 个 A 和第 iii 个 B 一定匹配．也就是说，如果我们让第 lll 个到第 rrr 个 A 在一个子序列中，那么第 lll 个到第 rrr 个 B 也要再这一子序列中，且能够匹配也就是说必须被移动到第 rrr 个 A 后面．

设 cti\mathrm{ct}_icti​ 为第 iii 个 A 前有多少个 B，那么划分出第 lll 个到第 rrr 个 A 造成的代价就是 ∑i=lrmax⁡{cti−(l−1),0}\sum\limits_{i = l}^r \max\{\mathrm{ct}_i - (l - 1), 0\}i=l∑r​max{cti​−(l−1),0}．

然后就可以 DP 了！设 fi,jf_{i, j}fi,j​ 表示前 iii 个 A 划分出 jjj 段的最小代价，转移枚举上次划分在那里，容易做到 O(n3)O(n^3)O(n3)．然后发现这个代价函数满足四边形不等式，整个问题就是凸的了！wqs 二分套个斜率优化即可．

C. Tourism

套路世界大战！

可以求出区间内每一个点到根节点的路径并大小，然后减去区间 LCA 到根的路径大小，就是区间虚树覆盖的点集大小．

区间 LCA 可以通过查询区间 dfn 最小值和最大值，求它们对应点的 LCA 解决；路径并大小是经典套路，拿一个 LCT 然后扫描线，access 一下新加的点，每次虚实边切换时撤销上次的贡献并插入当前的贡献，复杂度 O(nlog⁡2n)O(n \log^2 n)O(nlog2n)．

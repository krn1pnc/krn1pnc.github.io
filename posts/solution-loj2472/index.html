LOJ 2472 「九省联考 2018」IIIDX 题解

发表于 2023-06-09 分类于 题解

思路

容易观察到难度限制就是让我们构造小根堆，于是 did_idi​ 互不相同时贪心地给编号较小的子树分配较大的权值即可．记 sizeu\mathrm{size}_usizeu​ 代表 uuu 子树的大小．

did_idi​ 中存在相同元素时，上述贪心策略寄了．为什么？考虑一个点 uuu，在我们分配完成后，若我们能够将分配给 uuu 的子树中某个节点 vvv 的权值与分配给 uuu 的某个编号大于 uuu 的兄弟节点 xxx 的权值交换，我们就能够通过这样的调整来增大答案的字典序．当 did_idi​ 互不相同时，这样的 uuu 显然不存在；当 did_idi​ 中存在相同元素时，uuu 的子树中可以填和自己相同的值，而按照贪心策略则会尽量填大的值，这显然是不优的．

要求字典序最大，我们应当考虑按照 1∼n1 \sim n1∼n 的顺序按位确定权值．而我们又发现，如果我们已经确定完 1∼u−11 \sim u - 11∼u−1 所填的权值，决策节点 uuu 的权值时，若这个值选得太大，则子树中没有足够多的权值去填充．这说明节点 uuu 能填的权值存在一个上界，我们尝试去二分这个上界，贪心地选择它作为答案．

问题转化为判定性问题，我们想要判断的是，如果已经确定了 1∼u−11 \sim u - 11∼u−1 的权值，节点 uuu 上填 xxx 这个值后是否存在一个合法的填数方案．由于 1∼u−11 \sim u - 11∼u−1 的权值已经确定，那么直接与未确定的点相连的节点对其子树中所填的值都有限制．于是我们维护 cti\mathrm{ct}_icti​ 表示 iii 这个值还剩多少个，booki\mathrm{book}_ibooki​ 表示当前填 iii 这个值的所有节点子树中需要多少个大于等于 iii 的值．当 uuu 填 xxx 时，ctx\mathrm{ct}_xctx​ 会减去 111，bookx\mathrm{book}_xbookx​ 会减去 sizeu−1\mathrm{size}_u - 1sizeu​−1．由于我们已经确定了的点在树上构成一个联通块，子树限制必然不交，也就是说 ∑j≥xbookj\sum_{j \ge x} \mathrm{book}_j∑j≥x​bookj​ 就是当前所需 ≥x\ge x≥x 权值的个数．那么判断存在合法方案时，直接假设 uuu 填了 xxx，然后只需要判断 ct\mathrm{ct}ct 的后缀和是否处处不小于 book\mathrm{book}book 的后缀和即可．

暴力做的时间复杂度是 O(n2log⁡n)O(n^2 \log n)O(n2logn) 的，可以拿到和贪心一样的分数．

于是考虑如何优化这个二分过程，如果我们记 si=∑j≥icti−bookis_i = \sum_{j \ge i} \mathrm{ct}_i - \mathrm{book}_isi​=∑j≥i​cti​−booki​，每次确定 uuu 的值时对 sss 的影响形如前缀减 sizeu\mathrm{size}_usizeu​，而二分实际上时找到最大的 xxx，使得 ∀i≤x,si−sizeu≥0\forall i \le x, s_i - \mathrm{size}_u \ge 0∀i≤x,si​−sizeu​≥0．于是使用线段树维护 sss 的最小值，支持区间加法即可．

注意决策到 uuu 时需先撤销先前决策 ⌊u/k⌋\lfloor u / k \rfloor⌊u/k⌋ 时 uuu 子树对 book\mathrm{book}book 的影响．

二分可以在线段树上进行，时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)．

代码

#include <algorithm>
#include <cstdio>

inline int rd() {
	int x = 0, f = 1, c = getchar();
	while (((c - '0') | ('9' - c)) < 0)
		f = c != '-', c = getchar();
	while (((c - '0') | ('9' - c)) > 0)
		x = x * 10 + c - '0', c = getchar();
	return f ? x : -x;
}

const int N = 5e5;

int n, a[N + 5];
double k;
int b[N + 5], bct;

int fa[N + 5], siz[N + 5];
int val[N + 5], ct[N + 5], bk[N + 5];

#define lch (p * 2)
#define rch (p * 2 + 1)
#define mid ((cl + cr) / 2)
struct node {
	int mn, tg;
	void upd(int d) {
		mn += d, tg += d;
	}
} t[N * 4 + 5];
void pushup(int p) { t[p].mn = std::min(t[lch].mn, t[rch].mn); }
void pushdown(int p) {
	if (!t[p].tg) return;
	t[lch].upd(t[p].tg);
	t[rch].upd(t[p].tg);
	t[p].tg = 0;
}
void upd(int l, int r, int d, int p = 1, int cl = 1, int cr = bct) {
	if (cl == l && cr == r) return t[p].upd(d);
	pushdown(p);
	if (r <= mid) upd(l, r, d, lch, cl, mid);
	else if (l > mid) upd(l, r, d, rch, mid + 1, cr);
	else upd(l, mid, d, lch, cl, mid), upd(mid + 1, r, d, rch, mid + 1, cr);
	pushup(p);
}
int getpos(int k, int p = 1, int cl = 1, int cr = bct) {
	if (cl == cr) return cl - (t[p].mn < k);
	pushdown(p);
	if (t[lch].mn >= k) return getpos(k, rch, mid + 1, cr);
	else return getpos(k, lch, cl, mid);
}
#undef lch
#undef rch
#undef mid

int main() {
	n = rd(), scanf("%lf", &k);
	for (int i = 1; i <= n; i++) a[i] = b[i] = rd();
	std::sort(b + 1, b + 1 + n), bct = std::unique(b + 1, b + 1 + n) - b - 1;
	for (int i = 1; i <= n; i++) a[i] = std::lower_bound(b + 1, b + 1 + bct, a[i]) - b;

	for (int i = 1; i <= n; i++) upd(1, a[i], 1);
	for (int i = 1; i <= n; i++) siz[i] = 1, fa[i] = i / k;
	for (int i = n; i >= 1; i--) siz[fa[i]] += siz[i];

	val[0] = 1, upd(1, 1, -n);
	for (int u = 1; u <= n; u++) {
		upd(1, val[fa[u]], siz[u]);
		int ans = getpos(siz[u]);
		val[u] = ans;
		upd(1, ans, -siz[u]);
	}

	for (int i = 1; i <= n; i++) {
		printf("%d%c", b[val[i]], " \n"[i == n]);
	}
	return 0;
}

考虑构造一个 DAG 状的自动机，接受的字符串数就变成了 DAG 上的路径计数．然后就可以通过构造一定量的重边来减少状态数．

具体而言就是先挂一个有 666 个节点的链，然后对于距 111 号点距离为 iii 的节点，由它向终止节点连 ai+1a_{i + 1}ai+1​ 条边．注意到字符集大小是 202020，而 a5=42a_5 = 42a5​=42，出边数量不够用，那么由 444 往 555 连 333 条重边，这样 555 就只用往终止节点连 141414 条边了．然后类似的构造把 a6a_6a6​ 挂上就行．

赛场大佬们的乱搞是钦定停在 111 上，然后直接搜邻接矩阵．注意到我们知道所有测试点中 a6a_6a6​ 的值，那么打个表就能过去．

定义一个三元组 (i,j,k)(i, j, k)(i,j,k) 的价值为 ai+aj+aka_i + a_j + a_kai​+aj​+ak​．

假设在一次询问中，我们作为答案的三元组为 (i,j,k)(i, j, k)(i,j,k)．尝试观察能够作为答案的三元组的性质．

  • 如果 ai<aja_i < a_jai​<aj​，且存在 i<p<ji < p < ji<p<j 使得 ap>aia_p > a_iap​>ai​，显然 (p,j,k)(p, j, k)(p,j,k) 合法且更优．
  • 如果 ai>aja_i > a_jai​>aj​，且存在 i<p<ji < p < ji<p<j 使得 ap>aja_p > a_jap​>aj​，显然 (i,p,k)(i, p, k)(i,p,k) 合法且更优．

那么每个数可以与左 / 右边第一个比它大的数作为答案中的前两个，有用的前两个形成的对只有 O(n)O(n)O(n) 个．

考虑 从右到左 进行扫描线，对每个位置维护值 cic_ici​ 表示三元组 (x,y,i)(x, y, i)(x,y,i) 取到最大价值时，ax+aya_x + a_yax​+ay​ 的值，答案就是区间内 ai+cia_i + c_iai​+ci​ 的最大值．l→l−1l \to l - 1l→l−1 的过程中，考虑插入 ala_lal​ 造成的影响，将 lll 作为第一个元组的有用的对加入时，对 cic_ici​ 的影响形如后缀取 max⁡\maxmax．具体而言，设当前加入的对是 (l,i)(l, i)(l,i)，那么相当于将 [i+(i−l),n][i + (i - l), n][i+(i−l),n] 这段后缀对 al+aia_l + a_ial​+ai​ 取 max⁡\maxmax．

以下是一些民科内容，不保证正确性，建议不看．

注意此处从左到右扫描线不可做．因为若想从左到右进行扫描线，我们必定要在三元组的第一个数处计算贡献，不然区间查询就废掉了．这样的话我们就需要找到关于三元组中后两个元素的性质，然而注意到我们的不等号方向是 j−i≤k−jj - i \le k - jj−i≤k−j，考虑前两个数形成的对时，我们对答案做的调整是在放松这个不等式．但当我们尝试调整后两个数形成的对时，我们是在往紧的方向调整，然后性质就不好了．

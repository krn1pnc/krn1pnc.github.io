t+tai+bi+1+(t+tai+bi+1)aj+bj≤t+taj+bj+1+(t+taj+bj+1)ai+bi⇒bi+biaj+aj+bj≤bj+bjai+ai+bi\begin{alignedat}{} & & t + ta_i + b_i + 1 + (t + ta_i + b_i + 1)a_j + b_j &\le t + ta_j + b_j + 1 + (t + ta_j + b_j + 1)a_i + b_i \\ & \Rightarrow & b_i + b_ia_j + a_j + b_j &\le b_j + b_ja_i + a_i + b_i \end{alignedat} ​⇒​t+tai​+bi​+1+(t+tai​+bi​+1)aj​+bj​bi​+bi​aj​+aj​+bj​​≤t+taj​+bj​+1+(t+taj​+bj​+1)ai​+bi​≤bj​+bj​ai​+ai​+bi​​

很神奇啊，ttt 没了，也就是说对于选取的一个集合，我们去买东西的顺序其实是一定的．

将序列按照上述方式排序后，问题弱化为选出子序列，最大化子序列长度，容易设计一个 DP 来解决．设 fi,jf_{i, j}fi,j​ 表示考虑了前 iii 个商店，当前时刻为 jjj 的最长子序列长度，转移是简单的．

这 DP 第二维有 10910^9109，显然寄了，但是注意到 ai≠0a_i \not= 0ai​=0 时的答案并不大，大概是 log⁡\loglog 级别的（这是因为每选一个 ai≠0a_i \not= 0ai​=0 的商店时，时刻至少翻倍），于是考虑只 DP ai≠0a_i \not= 0ai​=0 的部分，剩下的商店显然可以贪心拼接，然后将 DP 的值域和定义域换一下，设 fi,jf_{i, j}fi,j​ 表示考虑前 iii 个商店，子序列长度为 jjj 的最小时刻，就可以解决了．

时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)．

B. 下午茶

由于 ppp 是奇数，我们建出 x→2x mod nx \to 2x \bmod nx→2xmodn 的边，容易证明这张图由若干个环构成．

又由于 (2y+1,p)=1(2^y + 1, p) = 1(2y+1,p)=1，也就是说不存在 kkk 使得 2k≡−1(modp)2^k \equiv -1 \pmod p2k≡−1(modp)，这证明 xxx 和 −x-x−x 一定在不同的环中．且 xxx 和 −x-x−x 所在的环中的元素仅相差一个符号，也就是说可以将其一一对应，我们的目标是用最小的代价使得对应元素相等．

上文中可以看出，环的总数为偶数，所以我们只需要解决有两个环的情况就解决了问题．

考虑将环对齐后对位相减，目标变成调整成 000，这不是我们 糖果传递 吗！然后就做完了．

B. Graph

构造．/qd

先尝试一下树的部分分！

我们考虑以下的构造方案：

  1. 从一个点开始，对树进行 dfs．初始所有点都在 BBB 集合中．

  2. 搜到 uuu 时，将 uuu 从 BBB 移动到 ppp 中．

  3. 从 uuu 离开时，将 uuu 从 ppp 移动到 AAA 中．

  4. 若某次操作后，∣A∣=∣B∣|A| = |B|∣A∣=∣B∣，那么我们就得到了一组合法的构造．

每次操作都会使得 ∣B∣−∣A∣|B| - |A|∣B∣−∣A∣ 减少 111，而 dfs 前 ∣B∣−∣A∣=n|B| - |A| = n∣B∣−∣A∣=n，完成 dfs 后 ∣B∣−∣A∣=−n|B| - |A| = -n∣B∣−∣A∣=−n，也就是说一定存在某次操作，操作完后 ∣A∣=∣B∣|A| = |B|∣A∣=∣B∣，不存在无解．

考虑扩展到图上，发现这个做法的唯一阻碍在于，若我们搜出的生成树中有横叉边，那么 A,BA, BA,B 集合间就可能有边．然而如果我们直接选取一棵 dfs 树，那么就解决了这个问题．
